ERC Marsyard Rover Module-Based
Benchmarking Criteria
Benchmark structure divided into Perception, SLAM/Mapping, Path Planning, Control, and Mission
Evaluation for a ROS 2 Marsyard rover testing system.

Purpose
This document defines how to evaluate the complete Marsyard rover navigation system using clear module-based
benchmarks. Each technical module receives its own measurable score, then all scores are combined into a full system
benchmark. The structure is designed for a testing team that already has a Marsyard world and rover model and needs
objective testing criteria.

Top-Level System Flow
Scenario YAML -> Marsyard simulation -> Rover sensors -> Perception -> SLAM/Mapping -> Path Planning
-> Control -> Mission Supervisor -> Evaluator -> Final Report

Full System Score = 0.20 x Perception + 0.20 x SLAM/Mapping + 0.25 x Path Planning + 0.25 x Control +
0.10 x Mission/Evaluation

1. Perception Benchmark
Perception is responsible for converting raw sensor data into useful information for navigation. In the Marsyard system,
perception should detect obstacles, identify risky terrain, validate sensor frames, and provide data that can support
heightmap, traversability, and costmap generation.

Inputs
● /scan or /pointcloud
● /camera/depth or stereo point cloud if available
● /imu
● /tf
● Marsyard world ground truth for validation

Outputs
● Detected obstacles
● Terrain point cloud
● Obstacle / terrain classification
● Sensor validity report

2. SLAM / Mapping Benchmark
SLAM/Mapping is responsible for estimating the rover pose and building or maintaining the map used by navigation. If the
system uses a static Marsyard map, this module still measures localization accuracy, map alignment, drift, and
heightmap/costmap consistency.

Inputs
● /odom
● /imu
● /tf
● LiDAR/depth data
● static map or heightmap
● ground truth pose from simulation if available

Outputs
● Estimated rover pose
● Map -> odom -> base_link transform
● Updated map or local map
● Heightmap/costmap alignment result
● Localization confidence/status

3. Path Planning Benchmark
Path planning is responsible for creating a valid route from the rover position to the current checkpoint or finish point. In
this Marsyard benchmark, the planner must consider obstacles, rough terrain, slopes, and the costmap produced from
heightmap/traversability data.

Inputs
● Current rover pose
● Checkpoint or finish goal
● Global costmap
● Traversability map
● Robot footprint and limits
● Reference path for evaluation

Outputs
● Global path
● Path cost
● Planning status
● Replanned path when blocked

What is calculated
Metric

How to Calculate

<= 2 sec preferred

0

<= 1.35

Replanning Success

>= 85%

Path Planning Score
Planning Score = 0.25 x PlanningSuccess + 0.15 x PlanningTimeScore + 0.25 x ObstacleAvoidance + 0.15 x
PathCostScore + 0.10 x PathLengthScore + 0.10 x ReplanningScore

Pass condition: Path Planning Score >= 85/100.

4. Control Benchmark
Control is responsible for converting the planned path into motion commands. The controller should track the path, avoid
local obstacles, keep the rover stable, and generate smooth /cmd_vel commands.

5. Mission / Evaluation Benchmark
Mission/Evaluation verifies that the benchmark runner works correctly. It checks that checkpoints are sent in order, finish
is detected, logs are saved, and final scoring is reproducible across trials.

Inputs
● Scenario YAML
● Checkpoint list
● Rover pose and trajectory
● Collision status
● Time
● Module scores

Outputs
● Checkpoint progress
● Final mission status
● CSV/JSON logs

6. Per-Scenario Benchmark Score
Each scenario should generate one final score based on the five module scores. This makes it easy to compare routes
such as right-side checkpoints, left-side checkpoints, center route, slope route, rough-terrain route, and full ERC-like route.
Scenario Score = 0.20 x Perception + 0.20 x SLAM/Mapping + 0.25 x PathPlanning + 0.25 x Control + 0.10
x Mission/Evaluation

7. Full System Benchmark
The full system benchmark is calculated over all scenarios and trials. It should summarize the performance of the
complete simulation/testing framework, not only a single run.
Scenario Performance Score = average(all scenario scores)

Full System Score = average scenario performance, with failure-rate penalties applied if needed.

10. Minimum Acceptance Criteria
● Perception Score >= 85
● SLAM / Mapping Score >= 85
● Path Planning Score >= 85
● Control Score >= 85
● Mission / Evaluation Score >= 90
● Full System Score >= 80
● Collision Rate <= 10%
● Timeout Rate <= 20%
● Average Checkpoint Completion >= 80%

Core Statement
This benchmark evaluates the Marsyard rover navigation system by separating the system into Perception,
SLAM/Mapping, Path Planning, Control, and Mission Evaluation. Each module has its own measurable score, and the
final benchmark combines all module scores across all scenarios to produce one complete system score.
